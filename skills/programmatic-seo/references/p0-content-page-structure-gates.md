# P0 内容页结构门槛

## 使用时机

确定内容页 Brief 后，开始规划 H1、H2、模块、内链、CTA、图片和页面结构前，必须读取本文件。结构的目标不是套模板，而是让用户最快得到答案，让搜索引擎看懂主题关系，让页面自然承接 Tomako 的工具和产品价值。

## 结构总原则

内容页必须满足：

- 第一屏说明这篇页面要解决什么问题。
- 前 20% 内容给出可用答案或清晰路线，不绕背景。
- 每个 H2 对应一个真实搜索问题、决策点、步骤、标准、误区、例子或下一步。
- 每个 section 都有独立信息价值，不能只是换句话重复引言。
- 站内内链有清楚上下游关系，不做孤立文章。
- 结尾不是简单 CTA，而是把用户自然带到工具、模板、相关指南或下一步。

## 通用页面骨架

大多数内容页可从以下骨架裁剪：

1. H1：直接匹配用户任务或核心问题。
2. Intro：说明适合谁、读完能得到什么、适用边界。
3. 快速答案 / 结论摘要：先回答，不把答案藏到最后。
4. 主体模块：按页面类型展开步骤、标准、对比、模板、案例、排查或说明。
5. 示例或可复制材料：让内容从“讲道理”变成“可使用”。
6. 常见误区 / 风险边界：消除疑虑，避免错误使用。
7. 与 Tomako 的关系：推荐相关工具、模板或工作流下一步。
8. FAQ：覆盖真实搜索疑问，不堆关键词。
9. 相关内容 / 下一步：内部链接到 hub、cluster、Tool、Template。

不要把每篇都写成一样的九段。选择和搜索意图匹配的模块。

## 页面类型结构

### How-to / Tutorial

适合“怎么做”“如何完成某任务”。

必须包含：

- 任务适用场景。
- 准备材料或输入。
- 关键步骤。
- 可复制示例或模板。
- 常见错误与修复。
- 用 Tomako 工具继续完成的入口。

避免：长背景、泛泛理论、没有示例的步骤。

### Template / Checklist

适合“模板”“清单”“brief”“example”。

必须包含：

- 模板能解决什么工作。
- 谁适合用。
- 模板字段解释。
- 填写示例。
- 复制/下载/生成入口。
- 什么时候需要人工复核。

避免：只展示模板名称和几段介绍，没有可用字段或示例。

### Comparison / Alternatives / Versus

适合“哪个好”“A vs B”“替代方案”。

必须包含：

- 用户决策标准。
- 对比对象和适用场景。
- 选择建议。
- Tomako 的适合/不适合边界。
- 事实来源或明确推断。

避免：无证据踩竞品、编造排名、把广告文写成客观评测。

### Best / List / Directory

适合“best X”“top X”“工具列表”。

必须包含：

- 入选标准。
- 每项适合谁。
- 核心优缺点。
- 选择路径。
- 更新机制或数据来源。

避免：只堆列表、不说明为什么、没有筛选标准。

### Glossary / Explainer

适合“what is”“是什么意思”“概念解释”。

必须包含：

- 一句话定义。
- 通俗例子。
- 为什么重要。
- 常见误解。
- 与相邻概念区别。
- 如何在 Tomako 工作流中使用。

避免：百科式长定义、没有场景、没有下一步。

### Troubleshooting

适合“为什么失败”“无法生成”“下载不了”“不显示结果”。

必须包含：

- 现象分类。
- 可能原因。
- 逐步排查。
- 能解决和不能解决的边界。
- 何时重试、换输入或联系支持。

避免：只写“请重试”，没有原因和恢复路径。

### Hub / Pillar

适合大主题入口和内容集群。

必须包含：

- 主题地图。
- 核心概念和路线。
- 下游 cluster 链接。
- 工具/模板入口。
- 更新计划或维护方式。

避免：把 hub 写成超长孤立文章；hub 的价值在组织关系和分发入口。

## 内容集群结构

批量内容必须先规划集群，不要一次生成孤立页面。

建议输出：

```text
主题集群：
Hub 页面：
Cluster 页面：
每个页面的主关键词：
每个页面的搜索意图：
每个页面的独立角度：
Hub -> Cluster 内链：
Cluster -> Hub 内链：
Cluster 之间的横向内链：
相关 Tool / Template：
优先级：
```

如果多个页面的 H1、搜索意图、读者阶段和 CTA 都接近，说明存在重复或关键词蚕食风险，应合并或重新定位。

## 反薄内容检查

以下情况视为结构失败：

- 只是把关键词替换到同一个模板里。
- H2 看起来很多，但每段都在讲同一个泛概念。
- 页面没有示例、清单、判断标准、数据、具体步骤或独立观点。
- FAQ 能回答的内容被拉成长正文。
- 与 Tomako 没有产品、工具、模板或工作流连接。
- 站内没有相关内链，也没有进入该页的合理路径。
- 只服务搜索引擎，不帮助真实用户做决定或完成任务。

## 模块取舍

内容页模块可以自由，但必须能标注作用：

- **答案型**：直接回答搜索问题。
- **判断型**：帮助用户比较、选择、避坑。
- **操作型**：给步骤、模板、示例、清单。
- **信任型**：说明来源、经验、边界、更新。
- **转化型**：引导使用工具、模板或下一步。

一个模块如果无法归类，通常应该删除或并入其他模块。

## CTA 和内链位置

内容页 CTA 不只放结尾。可以在这些位置自然出现：

- 用户刚理解问题后，给相关工具入口。
- 用户读完步骤后，给模板或生成器入口。
- 用户遇到边界后，给排查页或支持路径。
- 用户读完对比后，给最适合的下一步。

CTA 文案必须描述真实动作，例如“生成上线清单”“打开模板”“分析页面”“继续阅读对比”。不要写泛泛的“了解更多”。
