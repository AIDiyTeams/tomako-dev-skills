# P0 内容页开工 Brief Gate

## 使用时机

新建、重写、批量规划、诊断或校准 Tomako Blog / Template / MCP 等内容型 SEO 页面时，必须读取本文件。内容页不是 Tool 页的替代品；它负责承接搜索问题、建立信任、解释场景、补足决策信息，并把用户导向工具、模板或下一步。

## 开工前必须确认

内容页开工前，至少确认以下信息，并在对话框返回简短摘要：

- 页面类型：教程、清单、模板说明、对比、最佳实践、术语解释、场景页、问题排查、案例/经验、集群 hub、cluster article。
- 目标读者：谁会搜索它，他处在哪个阶段。
- 搜索意图：用户想学习、判断、比较、下载模板、解决问题，还是找下一步工具。
- 目标市场和语言：`zh`、`en`，目标地区或默认全球。
- 主关键词和候选词：先用上下文收敛，再按 `p0-content-gates.md` 低成本验证。
- 页面承诺：读完后用户能知道什么、拿到什么、决定什么或继续做什么。
- 独立信息增益：这篇内容相比普通搜索结果、AI 泛文、竞品文章或 Tomako 其他页面新增什么价值。
- 与产品关系：要链接到哪个 Tool、Template、功能页、内容集群或 CTA。
- 证据来源：用户提供、Tomako 产品事实、公开资料、DataForSEO、SERP、竞品、经验判断。
- 风险边界：是否涉及法律、隐私、合规、财务、医疗、安全、平台规则或竞品事实。
- 发布形态：Blog / Template / MCP / 新栏目；是否双语同步；是否进入 sitemap 和索引。

## 页面类型判断

不要先决定“写一篇 blog”，再硬塞内容。先看搜索意图，再选页面类型：

| 搜索意图 | 更适合的页面类型 | 重点 |
| --- | --- | --- |
| 想马上解决步骤问题 | How-to / Tutorial | 直接步骤、示例、常见错误、下一步工具 |
| 想判断哪种方案好 | Comparison / Alternatives | 选择标准、适用场景、取舍、Tomako 位置 |
| 想要可复制材料 | Template / Checklist | 可用模板、字段说明、使用方式、下载/复制入口 |
| 想理解概念 | Glossary / Explainer | 定义、例子、误区、相关概念、工具承接 |
| 想找一组方案 | Best / List / Directory | 筛选标准、推荐理由、适用人群、差异 |
| 遇到失败或异常 | Troubleshooting | 现象、原因、解决步骤、边界、替代路径 |
| 需要完整主题入口 | Hub / Pillar | 主题地图、子主题链接、核心指南、后续路线 |

如果搜索意图明显是“使用工具完成任务”，优先考虑 Tool Page，而不是纯内容页。

## 内容集群和内链判断

内容页不能孤立上线。开工前必须判断它在站点结构中的位置：

- 它是否属于某个主题集群，例如 launch、SEO、模板、品牌、Agent、工具使用。
- 它是否应该链接到一个相关工具页，例如生成器、分析器、下载器、检查器。
- 它是否应该链接到一个模板页，例如 launch brief、social post kit。
- 它是否应该作为 hub 收拢多个 cluster，还是作为 cluster 支撑一个 hub。
- 它与已有内容是否抢同一个关键词；若抢，应合并、改角度或做内链分工。

建议每个内容页至少规划：

- 1 个上游 hub 或栏目入口。
- 2 到 5 个相关下游内容或工具。
- 1 个明确的产品/工具/模板 CTA。

内链必须服务用户下一步，不是为了凑数量。

## 关键词证据和成本控制

内容页同样适用 `p0-content-gates.md` 的关键词证据流程：

1. 先从用户需求、产品语境和页面类型生成少量候选词。
2. 在对话框返回候选小表。
3. 优先复用已有 `docs/seo-keyword-briefs/{slug}.md/.json`。
4. DataForSEO 查询数量按实际场景决定，不设固定默认数量；CLI 单次硬上限是 50 个关键词。
5. 根据 SERP 意图决定页面类型和结构，而不是只看搜索量。

不要在需求模糊时批量查询几十个关键词。不要因为某个词搜索量大，就写与 Tomako 产品无关的内容。

## 不可开工的情况

以下情况必须先追问或写明假设：

- 不知道读者是谁。
- 不知道用户搜索后想完成什么。
- 不知道页面要导向哪个 Tomako 工具、模板或产品价值。
- 只给了一个关键词，但没有页面类型、用户任务或差异化角度。
- 需要引用事实、竞品、数据或平台规则，但来源不清。
- 内容可能产生法律、财务、医疗、安全或合规误导。

## 内容页完成前必须能回答

一篇内容页在进入写作或实现前，必须能简短回答：

- 用户为什么会搜这篇内容。
- 这篇内容比普通 AI 回答或竞品文章多了什么。
- 用户看完能做什么决定或动作。
- Tomako 在这里提供什么价值。
- 这篇内容与站内哪些页面互相支撑。
